### General Settings

Set up the basic information and branding for your workspace.

<Tabs>
  <Tab title="Basic Info" icon="info">
    Configure workspace name, description, and contact information.
    
    <CodeGroup tabs="JSON,Environment Variables">
      ```json
{
  "name": "My Workspace",
  "description": "Documentation workspace for my team",
  "contactEmail": "admin@example.com",
  "timezone": "UTC",
  "language": "en"
}
      ```
      ```bash
export WORKSPACE_NAME="My Workspace"
export WORKSPACE_DESCRIPTION="Documentation workspace for my team"
export CONTACT_EMAIL="admin@example.com"
export TIMEZONE="UTC"
export DEFAULT_LANGUAGE="en"
      ```
    </CodeGroup>
  </Tab>
  <Tab title="Branding" icon="palette">
    Customize the look and feel of your documentation.
  </Tab>
  <Tab title="Notifications" icon="bell">
    Set up email and in-app notification preferences.
  </Tab>
</Tabs>

### Permission Levels

Dmitry Nesterenko Documentation supports granular permission controls for different user roles.

<ExpandableGroup>
  <Expandable title="Viewer Permissions" default-open="true">
    - Read access to all documentation
    - Search and browse content
    - Download and export documents
    - View change history
  </Expandable>
  <Expandable title="Editor Permissions" default-open="false">
    - All viewer permissions
    - Create and edit documents
    - Upload media files
    - Manage document versions
    - Comment on changes
  </Expandable>
  <Expandable title="Admin Permissions" default-open="false">
    - All editor permissions
    - Manage project settings
    - Add and remove team members
    - Configure integrations
    - Delete documents and projects
  </Expandable>
</ExpandableGroup>

## Integration Configuration

Connect your documentation workspace to external tools and services for enhanced functionality.

<Callout kind="info">
  Integrations allow you to sync documentation with your development workflow and automate updates.
</Callout>

### Version Control Integration

Link your projects to Git repositories for automatic synchronization and version control.

<CodeGroup tabs="GitHub,Self-hosted Git">
  ```bash
# Connect to GitHub repository
git remote add docs https://github.com/username/repo.git
git push docs main:docs
  ```
  ```bash
# Connect to self-hosted Git server
git remote add docs https://git.company.com/username/repo.git
git config --global credential.helper store
  ```
</CodeGroup>

Fine-tune your workspace and projects with advanced configuration options.

### Performance Tuning

Optimize your workspace for better performance based on your usage patterns.

<CodeGroup tabs="Configuration File,Environment Variables">
  ```json
{
  "cache": {
    "enabled": true,
    "ttl": 3600,
    "maxSize": "1GB"
  },
  "search": {
    "indexBatchSize": 1000,
    "maxResults": 10000
  },
  "uploads": {
    "maxFileSize": "50MB",
    "allowedTypes": ["md", "json", "yaml", "pdf"]
  }
}
  ```
  ```bash
export CACHE_ENABLED=true
export CACHE_TTL=3600
export SEARCH_BATCH_SIZE=1000
export MAX_UPLOAD_SIZE=50MB
  ```
</CodeGroup>

### Custom Domains

Set up custom domains for your documentation to match your brand.

<Steps>
  <Step title="Purchase Domain" icon="globe">
    Acquire a domain name from a registrar.
  </Step>
  <Step title="Configure DNS" icon="settings">
    Point your domain to our servers using CNAME records.
  </Step>
  <Step title="SSL Certificate" icon="shield">
    Enable HTTPS with automatic SSL certificate provisioning.
  </Step>
  <Step title="Verify Setup" icon="check">
    Test your custom domain and ensure all redirects work correctly.
  </Step>
</Steps>
