## Authentication

All API requests require authentication. Use API tokens or OAuth for secure access.

<Callout kind="info">
  Generate API tokens from your account settings. Keep tokens secure and rotate them regularly.
</Callout>

### API Token Authentication

Include your API token in the Authorization header for all requests.

<CodeGroup tabs="curl,JavaScript,Python">
  ```bash
curl -H "Authorization: Bearer YOUR_API_TOKEN" \
     -H "Content-Type: application/json" \
     https://api.docs.example.com/v1/projects
  ```
  ```javascript
const headers = {
  'Authorization': 'Bearer YOUR_API_TOKEN',
  'Content-Type': 'application/json'
};

fetch('https://api.docs.example.com/v1/projects', { headers })
  .then(response => response.json())
  .then(data => console.log(data));
  ```
  ```python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_TOKEN',
    'Content-Type': 'application/json'
}

response = requests.get('https://api.docs.example.com/v1/projects', headers=headers)
print(response.json())
  ```
</CodeGroup>

## Webhooks

Set up webhooks to receive real-time notifications about changes in your documentation.

<Callout kind="tip">
  Webhooks allow you to integrate documentation updates with your CI/CD pipeline or notification systems.
</Callout>

### Configuring Webhooks

Create webhooks to listen for specific events in your workspace.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Navigate to workspace settings and create a new webhook.
  </Step>
  <Step title="Set URL" icon="link">
    Provide the endpoint URL where notifications should be sent.
  </Step>
  <Step title="Choose Events" icon="filter">
    Select which events should trigger the webhook (document created, updated, etc.).
  </Step>
  <Step title="Test Webhook" icon="check-circle">
    Send a test payload to verify your endpoint is working.
  </Step>
</Steps>

### Webhook Payloads

Webhook payloads contain detailed information about the event that triggered them.

<CodeGroup tabs="Document Created,Project Updated">
  ```json
{
  "event": "document.created",
  "timestamp": "2024-10-15T10:30:00Z",
  "data": {
    "id": "doc_123",
    "title": "New API Guide",
    "projectId": "proj_456",
    "authorId": "user_789",
    "createdAt": "2024-10-15T10:30:00Z"
  }
}
  ```
  ```json
{
  "event": "project.updated",
  "timestamp": "2024-10-15T11:00:00Z",
  "data": {
    "id": "proj_456",
    "name": "Updated Project Name",
    "description": "New project description",
  ```

</CodeGroup>

Install the appropriate SDK for your programming language.

<CodeGroup tabs="npm,pip">
  ```bash
npm install @dmitrynesterenko/docs-sdk
  ```
  ```bash
pip install dmitrynesterenko-docs-sdk
  ```
</CodeGroup>

### Using the SDK

Initialize the SDK with your API token and start making requests.

<CodeGroup tabs="JavaScript,Python">
  ```javascript
const { DocsClient } = require('@dmitrynesterenko/docs-sdk');

const client = new DocsClient({
  token: 'YOUR_API_TOKEN',
  baseUrl: 'https://api.docs.example.com/v1'
});

// List all projects
client.projects.list().then(projects => {
  console.log(projects);
});
  ```
  ```python
from docs_sdk import DocsClient

client = DocsClient(
    token='YOUR_API_TOKEN',
    base_url='https://api.docs.example.com/v1'
)

# List all projects
projects = client.projects.list()
print(projects)
  ```
</CodeGroup>

## Error Handling

Handle API errors gracefully with appropriate status codes and error messages.

<Callout kind="danger">
  Always check response status codes and handle errors in your application code.
</Callout>

### Common Error Codes

| Status Code | Description |
|-------------|-------------|
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid or missing authentication |
| 403 | Forbidden - Insufficient permissions |
| 404 | Not Found - Resource does not exist |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error - Server-side issue |

### Error Response Format

All errors follow a consistent JSON format for easy handling.

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The request parameters are invalid",
    "details": {
      "field": "title",
      "issue": "Title is required"
    }
  }
}
```
